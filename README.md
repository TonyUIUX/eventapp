# KochiGo

A hyper-local event discovery app for Kochi, Kerala. Built with **Flutter + Firebase**.

## рџљЂ Setup Instructions

### Step 1 вЂ” Enable Windows Developer Mode (one-time)
> Required for Flutter symlinks on Windows
```
Settings в†’ Developer Mode в†’ ON
```
Then run:
```bash
flutter pub get
```

### Step 2 вЂ” Firebase Setup
1. Go to [Firebase Console](https://console.firebase.google.com) в†’ Create project `kochigo-app`
2. Add Android app with package name `com.kochigo.kochigo_app`
3. Download `google-services.json` в†’ place in `android/app/`
4. Enable **Firestore** (region: `asia-south1`)
5. Enable **Firebase Storage** (same region)

### Step 3 вЂ” FlutterFire Configure (generates firebase_options.dart)
```bash
dart pub global activate flutterfire_cli
flutterfire configure --project=kochigo-app
```
This **replaces** the placeholder `lib/firebase_options.dart`.

### Step 4 вЂ” Firestore Security Rules
In Firebase Console в†’ Firestore в†’ Rules:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /events/{eventId} {
      allow read: if true;
      allow write: if false;
    }
  }
}
```

### Step 5 вЂ” Firestore Composite Index
Firebase Console в†’ Firestore в†’ Indexes в†’ Add Index:
- Collection: `events`
- Fields: `isActive` (Ascending) + `date` (Ascending)

### Step 6 вЂ” Add Sample Data
Add test events to Firestore `events` collection. See `app_files_extracted/07_FIREBASE_SETUP_AND_SAMPLE_DATA.md`.
> вљ пёЏ Set the `date` field to **today or this weekend** so events appear in the app.

### Step 7 вЂ” Run
```bash
flutter run
```
